    2. Create `src/codagent/tools/shell.py` with the core execution function:

    `execute_command(command: str, timeout: int = 120) -> str`:
    - Uses `asyncio.create_subprocess_shell(command, stdout=PIPE, stderr=PIPE)` — NEVER `subprocess.run()` (blocks the event loop, prevents Ctrl-C handling).
    - Uses `asyncio.wait_for(proc.communicate(), timeout=timeout)` for timeout.
    - On `TimeoutError`: calls `proc.kill()`, then `await proc.wait()` to reap the process, returns `"[TIMEOUT] Command timed out after {timeout}s and was killed."`.
    - Combines stdout and stderr: stdout first, then `\n[stderr]\n{stderr}` if stderr is non-empty.
    - Decodes bytes with `errors='replace'` to handle binary output gracefully.
    - Returns formatted string: `"Exit code: {returncode}\n{output}"`.

    Truncation (SHEL-02):
    - Define `TRUNCATION_LIMIT = 10_000` as a module constant.
    - After combining stdout+stderr, if `len(output) > TRUNCATION_LIMIT`: truncate and append `"\n\n... [output truncated at {TRUNCATION_LIMIT} chars, {total} chars total]"`.
    - The truncation marker tells the model output was cut so it can request specific portions.

    `is_dangerous(command: str) -> bool`:
    - Define `DANGEROUS_PATTERNS` list of compiled regex patterns (from RESEARCH.md):
      - `rm -rf /` or `~` or `*` (both `-rf` and `-fr` flag orders)
      - `DROP TABLE`, `DROP DATABASE`
      - `DELETE FROM ... ;` without WHERE
      - `git push --force` or `-f`
      - `mkfs.`
      - `dd if=`
      - `> /dev/sd`
      - Fork bomb pattern
    - Returns True if any pattern matches (case-insensitive).

    Use `re.compile()` for the patterns at module level for efficiency.
  </action>

  <action>
    Add the approval gate and create the final `shell_tool` function that will be registered on the Pydantic AI agent.

    1. Create `prompt_user_approval(command: str, reason: str | None = None) -> bool`:
       - Uses `asyncio.get_event_loop().run_in_executor(None, ...)` to run `input()` in a thread (non-blocking — keeps event loop responsive for Ctrl-C). Per RESEARCH.md Pitfall 7.
       - Displays: `"[command] {command}"` on one line.
       - If reason provided, displays: `"[reason] {reason}"` on the next line.
       - Prompts: `"Approve? [Y/n] "` — Enter (empty input) defaults to yes per user decision.
       - Returns True if user enters empty, 'y', 'Y', 'yes', 'YES'. Returns False otherwise.

    2. Create `async def shell_tool(command: str) -> str`:
       - This is the function that will be decorated with `@agent.tool_plain` in agent.py.
       - The function docstring must be clear for the model: `"""Execute a shell command. Returns stdout, stderr, and exit code. Use this to interact with the filesystem, run tests, install packages, and perform any CLI operation."""`

       Logic flow:
       a. **Always check dangerous first** (even in yolo mode per user decision):
          - If `is_dangerous(command)`: call `prompt_user_approval(command, reason="Dangerous command detected — requires explicit approval")`.
          - If user rejects: return `"Command rejected by user. Dangerous command was not executed. Ask the user what they'd like to do instead."`

       b. **Check approval mode** (from `get_settings().mode`):
          - If mode is `"approval"`: call `prompt_user_approval(command)`.
          - If user rejects: return `"Command rejected by user. The user chose not to run this command. Ask the user what they'd like to do instead."`

       c. **Execute**: call `execute_command(command, timeout=get_settings().command_timeout)` and return the result.

    3. The rejection messages explicitly tell the model to ask the user what to do instead — per user decision: "On rejection, agent stops the current run and asks the user what they'd like to do instead."

    NOTE: The `shell_tool` function is NOT decorated here. It's a plain async function. The `@agent.tool_plain` decorator will be applied in `agent.py` when creating the Agent. This keeps the tool module independent of the agent framework for testability.
  </action>
