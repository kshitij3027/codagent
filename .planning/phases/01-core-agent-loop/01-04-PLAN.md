  <name>Task 2: Fix Ctrl-C at idle prompt to exit cleanly without traceback</name>
  <files>src/codagent/signals.py</files>
  <action>
Modify the _handle_sigint function in src/codagent/signals.py to fix the UAT Test 9 failure. The root cause is that `raise SystemExit(0)` cannot cleanly shut down because the input() thread running in ThreadPoolExecutor via run_in_executor blocks Python's shutdown sequence (it tries to join the thread, which is blocked on stdin, causing a deadlock and traceback).

Make the following specific changes to the idle branch (the `else` clause in _handle_sigint):

1. Replace `raise SystemExit(0)` with:
   ```python
   import os
   print("\nGoodbye.")
   os._exit(0)
   ```

2. Move the `import os` to the top of the file (module-level import alongside asyncio and signal).

The `os._exit(0)` call terminates the process immediately at the OS level, bypassing:
- asyncio.run() shutdown sequence (which tries to join the blocked input thread)
- Python's atexit handlers (where threading._shutdown deadlocks)
- The concurrent.futures thread pool shutdown

This is the correct fix because:
- Phase 2 will replace run_in_executor+input() with prompt-toolkit's prompt_async(), which does not have this issue (prompt_async is fully async, no blocking thread).
- Using os._exit(0) is safe here because there is no state to persist at idle -- no files to flush, no database connections, no pending writes.
- The "Goodbye." message is printed BEFORE os._exit() so the user sees a clean exit message.

Do NOT modify main.py -- the existing try/except (KeyboardInterrupt, SystemExit) block in main() is still useful as a fallback for edge cases.
