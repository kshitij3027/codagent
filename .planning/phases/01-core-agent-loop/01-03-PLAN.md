  <action>
    **1. Create `src/codagent/conversation.py`:**

    A simple `ConversationHistory` class that wraps a list of Pydantic AI messages:
    - `__init__(self)`: initialize empty `self._messages: list = []`
    - `get(self) -> list | None`: return `self._messages` if non-empty, else `None` (Pydantic AI expects `None` for first turn, not empty list)
    - `update(self, messages: list)`: replace `self._messages` with the new message list (from `result.all_messages()`)
    - `clear(self)`: reset to empty list (for future `/new` command in Phase 3)
    - `turn_count(self) -> int`: return count of user messages in history (useful for display)

    **2. Create `src/codagent/agent.py`:**

    `create_agent(model_string: str) -> Agent`:
    - Creates a `pydantic_ai.Agent` with:
      - `model=model_string` (from models.py registry)
      - `system_prompt` — a carefully crafted prompt that follows user decisions:
        ```
        You are a coding agent. You help users with programming tasks by executing
        shell commands. You have a single tool: `shell` — use it to interact with
        the filesystem, run tests, install packages, and perform any CLI operation.

        Behavior:
        - Be concise. You are a senior dev pairing with the user, not a tutorial.
        - If a request is ambiguous, ask for clarification before acting.
        - On command failure (non-zero exit), analyze the error and try a different
          approach. Give up after 3 failed attempts and explain what went wrong.
        - After completing a multi-step task, provide a brief summary of what was done.
        - Think step-by-step about what commands to run, but keep explanations short.
        ```
      - Register the shell tool from `codagent.tools.shell` using `@agent.tool_plain` decorator on a wrapper that calls `shell_tool(command)`. The tool docstring is critical — it's what the model sees to understand the tool.

    `async def run_agent_turn(agent: Agent, prompt: str, history: ConversationHistory) -> str`:
    - Calls `agent.run(prompt, message_history=history.get())`.
    - Updates `history.update(result.all_messages())`.
    - Returns `result.output` (the model's text response).
    - NOTE: Using `agent.run()` (not `agent.iter()`) for Phase 1 simplicity — per RESEARCH.md Open Question 4. The approval gate is inside the tool function, so we don't need node-level visibility. Phase 2 can switch to `iter()` when Rich UI needs to render intermediate states.

    **3. Create `src/codagent/signals.py`:**

    `SignalState` class:
    - `agent_task: asyncio.Task | None = None` — set when agent is running, cleared when done.

    `setup_signal_handler(loop: asyncio.AbstractEventLoop, state: SignalState)`:
    - Uses `loop.add_signal_handler(signal.SIGINT, handler)` — NOT `signal.signal()` (which is not event-loop-safe, per RESEARCH.md).
    - Handler logic:
      - If `state.agent_task` is not None and not done → `state.agent_task.cancel()` (SGNL-01: abort current operation, return to prompt)
      - Else → `raise SystemExit(0)` (SGNL-02: exit program at idle prompt)

  <action>
    Create `src/codagent/main.py` — the entry point that ties everything together.

    **`async def async_main():`**

    1. **Startup:**
       - Call `load_settings()` to load .env and initialize config.
       - Get default model string via `get_default_model()`.
       - Create agent via `create_agent(model_string)`.
       - Create `ConversationHistory()` instance.
       - Create `SignalState()` instance.
       - Set up signal handler: `setup_signal_handler(asyncio.get_event_loop(), signal_state)`.
       - Print a startup banner (Claude's discretion on format — keep it minimal):
         ```
         codagent v0.1.0
         Model: {friendly_name} ({model_string})
         Mode: {mode}
         Type your request or Ctrl-C to exit.
         ```

    2. **REPL Loop (`while True`):**
       - Get user input. For Phase 1 (no prompt-toolkit), use `await asyncio.get_event_loop().run_in_executor(None, input, ">>> ")` to keep input non-blocking (per RESEARCH.md Pitfall 1 and 7).
       - If input is empty or whitespace: `continue`.
       - If input is `exit` or `quit`: break the loop.
       - Create agent task: `signal_state.agent_task = asyncio.create_task(run_agent_turn(agent, user_input, history))`.
       - `try: response = await signal_state.agent_task`
       - `except asyncio.CancelledError:` — print `"\n[interrupted]"` and continue the loop (SGNL-01: returns to prompt).
       - `except Exception as e:` — print the error, continue the loop (don't crash on model errors).
       - `finally: signal_state.agent_task = None` — clear the task so Ctrl-C at idle exits.
       - Print `response` (plain text for Phase 1).

    3. **Shutdown:**
       - Print `"Goodbye."` and exit cleanly.

    **`def main():`**
    - Synchronous entry point for the console script.
    - `try: asyncio.run(async_main())`
    - `except (KeyboardInterrupt, SystemExit): pass` — suppress traceback on Ctrl-C at idle.

    **Edge cases to handle:**
    - EOFError from input (piped input ends): break the loop.
    - The `run_in_executor` + `input()` pattern is a Phase 1 workaround. Phase 2 replaces with prompt-toolkit `prompt_async()`.
  </action>
