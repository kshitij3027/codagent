Create `src/codagent/display.py` with a `Display` class that owns a singleton `rich.console.Console` instance. Implement:

**Panel factory — 4 distinct panel types with vibrant, high-contrast colors (per user decision):**
- `user` panel: bright_cyan border, "❯ You" title, ROUNDED box
- `response` panel: bright_green border, "🧠 Assistant" title, ROUNDED box
- `tool_call` panel: bright_yellow border, "🛠️ Tool Call" title, HEAVY box
- `tool_output` panel: bright_magenta border, "📋 Output" title, ROUNDED box

All panels: `title_align="left"`, `expand=True`, `padding=(1, 2)` (comfortable spacing per user decision).

Method `show_panel(content: str, panel_type: str)` — renders a static panel to console. For `response` type, render content as `rich.markdown.Markdown` with `code_theme="monokai"` for syntax-highlighted code blocks. For other types, render as plain text.

**Thinking spinner:**
Method `show_spinner(label: str = "Thinking...")` — starts a `rich.live.Live` display with a spinner renderable (use `rich.spinner.Spinner` with "dots" style, styled in bold cyan with a "🧠" prefix). The spinner uses a Live context so it can be seamlessly replaced by the streaming response (per user decision: "direct replace from spinner to response").

Method `hide_spinner()` — transitions from spinner to response streaming by swapping the Live renderable (do NOT stop and restart Live — that causes flicker per Pitfall 2 from research). Instead, reuse the same `_live` instance and call `_live.update()` with the new response panel renderable.

**Implementation details:**
- Store panel style configs in a `PANEL_STYLES` dict at module level.
- The `Display` class holds `self.console`, `self._live` (current Live context or None), and `self._buffer` (accumulated text during streaming).
- `Console()` is created once in `__init__`. Never create multiple Console instances (anti-pattern from research).
- Use `from rich import box` for box styles.

<task type="auto">
  <name>Task 2: Add streaming response and streaming tool output methods</name>
  <files>src/codagent/display.py</files>
  <action>
Add streaming methods to the `Display` class created in Task 1:

**Token-by-token response streaming (DISP-01, DISP-03):**
- `start_response_stream()` — If `_live` is already active (spinner was showing), reuse it. Otherwise, create a new `Live` context. Set `_buffer = ""`, `_streaming = True`. The Live display shows an empty response panel initially. Use `refresh_per_second=12` (balances smoothness vs CPU per Pitfall 3 from research).
- `stream_token(token: str)` — Append token to `_buffer`, update the Live display with a response-styled panel containing `rich.text.Text(self._buffer)`. Do NOT render as Markdown during streaming (too expensive per Pitfall 6 from research). Use plain Text during streaming.
- `finish_response_stream()` — Stop the Live display. Print a final static response panel with `rich.markdown.Markdown(self._buffer, code_theme="monokai")` using `console.print()`. This gives the "streaming looks plain, final render is rich" effect (matching ChatGPT/Claude.ai behavior per research). Reset `_live = None`, `_buffer = ""`, `_streaming = False`.

**Line-by-line tool output streaming (DISP-03):**
- `start_tool_output_stream()` — Create a new `Live` context with an empty tool_output panel. Set `_tool_buffer = ""`.
- `stream_tool_line(line: str)` — Append line to `_tool_buffer`, update Live with a tool_output-styled panel containing the accumulated text.
- `finish_tool_output_stream()` — Stop Live, print final static tool_output panel, reset buffers.

**Key implementation details:**
- Both streaming modes use the same pattern: Live context with `transient=False` (keep final output visible) and panel-wrapped content.
- The `_build_panel(content, panel_type)` helper creates a Panel with the correct style from PANEL_STYLES, used by both static and streaming methods.
- For the spinner-to-response transition: `hide_spinner()` should set a flag so `start_response_stream()` knows to reuse the existing Live context instead of creating a new one. This avoids the flicker gap (Pitfall 2).
- Handle edge case: if `finish_response_stream()` is called with empty buffer, skip the final Markdown render (avoid rendering empty string as Markdown).
