```python
async def async_main() -> None:
    # -- Startup --
    settings = load_settings()
    model_string = get_default_model()
    agent = create_agent(model_string)
    history = ConversationHistory()
    signal_state = SignalState()
    display = Display()
    session = create_prompt_session(settings.history_path)

    # Configure shell tool to use display for streaming
    from codagent.tools.shell import set_display
    set_display(display)

    loop = asyncio.get_event_loop()
    setup_signal_handler(loop, signal_state)

    # Rich-styled startup banner
    friendly_name = settings.default_model
    display.console.print()
    display.console.print(f"  [bold bright_cyan]codagent[/bold bright_cyan] v{__version__}")
    display.console.print(f"  [dim]Model:[/dim] {friendly_name} ({model_string})")
    display.console.print(f"  [dim]Mode:[/dim] {settings.mode}")
    display.console.print(f"  [dim]Enter submits · Escape+Enter for newline · Ctrl-C to exit[/dim]")
    display.console.print()

    # -- REPL Loop --
    with patch_stdout():
        while True:
            try:
                user_input = await get_user_input(session)
            except (EOFError, KeyboardInterrupt):
                break

            if not user_input or not user_input.strip():
                continue

            stripped = user_input.strip()
            if stripped.lower() in ("exit", "quit", "/exit"):
                break

            # Show the user's prompt in a styled panel
            display.show_panel(stripped, "user")

            # Run agent turn with streaming display
            signal_state.agent_task = asyncio.create_task(
                run_agent_turn_streaming(agent, stripped, history, display)
            )

            try:
                response = await signal_state.agent_task
            except asyncio.CancelledError:
                display.console.print("\n[dim][interrupted][/dim]")
                continue
            except Exception as e:
                display.console.print(f"\n[bold red][error][/bold red] {e}")
                continue
            finally:
                signal_state.agent_task = None

    # -- Shutdown --
    display.console.print("[dim]Goodbye.[/dim]")
```

**Key changes from Phase 1 main.py:**
1. Replace `input(">>> ")` with `get_user_input(session)` — async, non-blocking, with history and completions
2. Replace `print(response)` with streamed response panels (already handled inside `run_agent_turn_streaming`)
3. Replace bare `print()` calls with `display.console.print()` using Rich markup
4. Add `patch_stdout()` context manager wrapping the REPL loop (Pattern 6 from research — prevents Rich from corrupting prompt-toolkit input)
5. Add `display.show_panel(stripped, "user")` to echo user input in a styled panel before agent processing
6. Call `set_display(display)` to configure the shell tool for streaming output and styled approval
7. Remove `_get_input()` helper function (no longer needed)
8. Remove `run_in_executor` workaround (prompt-toolkit handles async input natively)

**Import changes:**
- Add: `from codagent.display import Display`
- Add: `from codagent.input import create_prompt_session, get_user_input`

**Keep `main()` synchronous entry point unchanged** — it still wraps `async_main()` in `asyncio.run()`.

  <action>
Update the signal handler to work cleanly with prompt-toolkit instead of using the `os._exit(0)` workaround.

**Changes to `_handle_sigint()`:**
- **Tier 1 (agent running):** Keep as-is — `state.agent_task.cancel()` works correctly.
- **Tier 2 (idle at prompt):** Replace `os._exit(0)` with `raise SystemExit(0)`. This is now safe because prompt-toolkit's `prompt_async()` doesn't use `run_in_executor` with a blocking thread — it's properly async and cancellable. The `SystemExit` will propagate up, prompt-toolkit will clean up terminal state (raw mode, cursor), and the process will exit cleanly (per Pitfall 8 from research).

**Remove:**
- `import os` (no longer needed for `os._exit`)
- The comment explaining why `os._exit` was necessary

**Keep:**
- `import asyncio, signal`
- `SignalState` class unchanged
- `setup_signal_handler()` function signature unchanged
- `loop.add_signal_handler(signal.SIGINT, _handle_sigint)` pattern unchanged

**Add a brief comment** explaining that prompt-toolkit's async input eliminates the blocked-thread problem that required `os._exit` in Phase 1.
