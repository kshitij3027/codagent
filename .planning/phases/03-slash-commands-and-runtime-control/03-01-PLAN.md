  <name>Task 1: Create commands.py with all slash command handlers and dispatch</name>
  <files>src/codagent/commands.py, src/codagent/models.py</files>
  <action>
Create `src/codagent/commands.py` with these functions:

1. `handle_model(args: str, agent: Agent, settings: Settings, display: Display) -> None`:
   - If `args` is empty (bare `/model`): call `_show_model_list()` to display current model and all available models.
   - If `args` is a name: call `get_model(args.strip().lower())`. On success, set `agent.model = model_string` AND `settings.default_model = friendly_name` (both must update together per Pitfall 3 and Pitfall 5 from research). Print confirmation: `Switched to {name} ({model_string})` in bold green.
   - On `ValueError` (invalid name): print error in bold red listing available models.

2. `_show_model_list(settings: Settings, agent: Agent, display: Display) -> None`:
   - Iterate sorted `MODEL_REGISTRY.keys()`. For the active model (`name == settings.default_model`), print with a `>` arrow indicator and `(active)` dim label in bold bright_green. For others, print dimmed with no indicator. Per user decision: "make it obvious which model is currently active."

3. `handle_approval(settings: Settings, display: Display) -> None`:
   - If `settings.mode == "approval"`: print `Already in approval mode.` (dim).
   - Otherwise: set `settings.mode = "approval"`, print `Mode: approval -- commands require confirmation before execution` in bold green.

4. `handle_yolo(settings: Settings, display: Display) -> None`:
   - If `settings.mode == "yolo"`: print `Already in yolo mode.` (dim).
   - Otherwise: set `settings.mode = "yolo"`, print `Mode: yolo -- commands will auto-execute (dangerous commands still require approval)` in bold yellow. The dangerous-command guardrail reminder is a locked user decision.

5. `handle_new(history: ConversationHistory, display: Display) -> None`:
   - Call `history.clear()`. Print `Conversation cleared.` (dim). Do NOT recreate the agent (locked decision). No confirmation prompt (locked decision).

6. `handle_help(display: Display) -> None`:
   - Create two Rich Tables using `box.SIMPLE`, `header_style="bold cyan"`, `padding=(0, 2)`.
   - Commands table with columns "Command" (bold, no_wrap=True) and "Description". Rows: `/model [name]`, `/approval`, `/yolo`, `/new`, `/help`, `/exit`.
   - Keyboard Shortcuts table with columns "Shortcut" (bold) and "Action". Rows: Enter (Submit prompt), Escape+Enter (Insert newline), Ctrl-C (Cancel current / exit), Up/Down (Command history), Tab (Complete slash command).
   - Print a blank line, both tables, then a blank line.

7. `dispatch_slash_command(text: str, agent: Agent, settings: Settings, history: ConversationHistory, display: Display) -> bool`:
   - If text does not start with `/`, return False.
   - Split on first space: `command = parts[0].lower()`, `args = parts[1] if len(parts) > 1 else ""`.
   - Dispatch table mapping `/model`, `/approval`, `/yolo`, `/new`, `/help` to their handlers.
   - If command not in table, return False.
   - Call the matched handler, return True.

Use TYPE_CHECKING imports for Agent, Settings, ConversationHistory, Display to avoid circular imports. Use lazy imports inside handler functions for `get_model`, `list_models`, `MODEL_REGISTRY`, `Table`, `box`.

Also in `src/codagent/models.py`: Remove the two bare `print()` calls on lines 70 and 93 (`print(f"[model] Resolved...")` and `print(f"[model] Default model...")`). These are Phase 1 debug aids that will corrupt Rich output (Pitfall 1 from research). No code parses their output.

<task type="auto">
  <name>Task 2: Wire slash command dispatch into the REPL loop</name>
  <files>src/codagent/main.py</files>
  <action>
Modify `src/codagent/main.py` to dispatch slash commands BEFORE the user panel and agent turn. This is the critical integration point (Pitfall 2 from research: dispatch must happen before `display.show_panel(stripped, "user")` to prevent slash commands from appearing in styled panels or reaching the agent).

In the REPL loop, after the exit check (`if stripped.lower() in ("exit", "quit", "/exit"): break`) and BEFORE `display.show_panel(stripped, "user")`, add:

```python
# Handle slash commands before agent turn
if stripped.startswith("/"):
    from codagent.commands import dispatch_slash_command
    if dispatch_slash_command(stripped, agent, settings, history, display):
        continue
    # Unknown slash command -- show error, do NOT send to agent
    display.console.print(
        f"[bold red]Unknown command:[/bold red] {stripped.split()[0]}. "
        "Type /help for available commands."
    )
    continue
```

Both branches use `continue` -- handled commands skip the agent turn, and unknown slash commands show an error and also skip the agent turn. No slash-prefixed input should ever reach the LLM.

Add `from codagent.config import load_settings, get_settings` is not needed -- `settings` is already available as a local variable from `load_settings()` at the top of `async_main()`. The import of `dispatch_slash_command` is lazy (inside the if block) which is fine for a rarely-executed path.
